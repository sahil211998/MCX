Trade with logic, not emotion — every signal backed by data, pattern, and discipline.

Only six contracts: ALUMINI, ZINCMINI, SILVERM, CRUDEOILM, NATURALGAS, NATGASMINI. Does not guarantee profit. Click to load full history, intraday proxy (default 30m, optional 15m), long-term proxy (default 4h), swing targets; call time is stored. Intraday timing uses a Yahoo proxy; intraday prices in the app are scaled to MCX INR using live LTP or last MCX daily close.

How this maps to institutional-style signal tools

Many prop and futures desks layer VWAP, relative volume, and prior-day (PD) high/low/close with platforms that show order flow (footprint, delta, DOM). Examples in that space include vendor order-flow analytics and multi-confluence chart suites (often paired with data terminals). This project does not subscribe to paid institutional feeds or MCX L2/L3 tape—it adds the same family of price/volume structure metrics we can compute from public bhav, watch LTP, and Yahoo proxy bars (intraday + long-term), plus your existing EMA/RSI/ATR/MACD/Bollinger confluence.

    Recent stored calls

    Same underlying rows; switch tabs to focus on intraday (15–30m proxy execution levels) or long / swing (2–4h Yahoo proxy default, MCX-scaled; MCX daily still in desk context). Intraday % vs Daily context % use different bar sets; Blend % mixes them for the headline score.
    Entry / stop / target in each row are the fixed snapshot from when that call was stored (2 dp INR); they are not recalculated or “extended” later.
    Sorting and filters use call time (call_generated_at, else created_at), shown in IST, newest first by default.
    The auto worker wakes every AUTO_SIGNAL_INTERVAL_SECS (default 120s) but will not generate a new stored call if the latest intraday BUY/SELL call for that product is still unresolved (has not hit target/stop vs the current MCX watch LTP). Default behavior: AUTO_SIGNAL_ONLY_AFTER_RESOLUTION=1. On top of that, it also applies a daily cap (default one new row per product per calendar day in AUTO_SIGNAL_CALENDAR_TZ / INTRADAY_OUTCOME_TZ, usually IST). Raise the cap with AUTO_SIGNAL_MAX_NEW_CALLS_PER_PRODUCT_PER_DAY (e.g. 2). Disable the worker: AUTO_SIGNAL_ENABLED=0. Buttons Generate all six / Analyze (store) always write new rows (no cap).
    Intraday outcomes (target/stop vs LTP) refresh on a daily schedule — default 23:25 Asia/Kolkata — not after every signal write. Env: INTRADAY_OUTCOME_LOCAL_HOUR, INTRADAY_OUTCOME_LOCAL_MINUTE, INTRADAY_OUTCOME_TZ.
